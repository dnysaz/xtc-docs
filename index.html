01 Introduction

XtermChat is a lightweight, self-hosted chat system built for the terminal. Unlike third-party platforms, XtermChat stores nothing on external servers. You deploy the server on your own VPS, connect with the CLI client or a browser, and all messages stay on infrastructure you control.

It is designed for developers and sysadmins who want something simple that just works — without the overhead of Matrix, the enterprise focus of Mattermost, or the data concerns of Slack. It also ships with a full bot monitoring system and a web admin panel.

02 How it works

XtermChat has two parts: a server you deploy on a VPS, and a client your team installs. They communicate over a REST API with JSON payloads. The client also ships with a bot system and web admin panel.

architecture
┌──────────────────────────────────────────────┐ │ YOUR VPS (:8080) │ │ xtc-server Python + Flask + SQLite │ │ ┌─────────┐ ┌──────────┐ ┌───────┐ ┌─────┐ │ │ │ rooms │ │ messages │ │ users │ │bots │ │ │ └─────────┘ └──────────┘ └───────┘ └─────┘ │ │ xtc.db (single file) │ │ bot_runner.py (background) │ └─────────────────┬────────────────────────────┘ │ REST API HTTP / HTTPS ┌─────────┼──────────┐ │ │ │ ┌──────▼──┐ ┌────▼────┐ ┌───▼──────┐ │ CLI Mac │ │Web :5000│ │CLI Linux │ │ xtc-cli │ │ Admin │ │ xtc-cli │ └─────────┘ └─────────┘ └──────────┘
🐍
Backend

Python + Flask. SQLite3 auto-created. No external DB needed. Runs on 2 pip packages.

⌨️
CLI Client

Built with prompt_toolkit. Full TUI with sidebar, mention highlight, link detection, emoji, and copy.

🌐
Web Admin

Flask serves HTML at :5000. Full admin panel: rooms, chat, bot management, dashboard.

🤖
Bot System

Background process on VPS. Monitors CPU, RAM, disk, uptime, SSL, logs, and more. Alerts sent to any room.

04 Server setup

SSH into your VPS and follow these steps. The entire process takes about 5 minutes.

1

Clone the server

git clone https://github.com/dnysaz/xtc-server.git
cd xtc-server
2

Install dependencies

pip3 install flask flask-cors werkzeug psutil

psutil is required for the bot monitoring system.

3

Open port 8080

sudo ufw allow 8080 && sudo ufw enable
⚠️

If your VPS has a separate firewall panel (DigitalOcean, Vultr), also open TCP 8080 there.

4

Start the server

python3 server.py start
# [*] Server started in background (PID: 12345)
5

Verify

curl http://localhost:8080
# {"service": "XtermChat Gateway", "status": "online", "version": "1.1"}

File structure

xtc-server/
xtc-server/ ├── server.py Flask app, all API + bot routes ├── db.py Database init (users, rooms, messages, bots) ├── room.py Room CRUD, bcrypt password hashing ├── connection.py Message save and retrieval ├── bot_runner.py Bot background process (runs on VPS) ├── check_db.py Interactive database inspector ├── xtc.db SQLite database (auto-created) ├── server.log Output log (auto-created) └── server.pid Process ID file (auto-created)

10 Room management

# Public room
xtc create:room @general

# Private room with password
xtc create:room @team secretpassword

# Interactive mode
xtc create:room

Creator privileges

When you create a room, your hardware UUID is recorded as creator_pin. Only you (from the same machine) can delete it or purge its messages.

⚠️

:purge shows a confirmation modal before deleting. It permanently removes all messages from the server. The room itself remains. Cannot be undone.

12 Bot system

XtermChat has a built-in server monitoring bot system. Bots run as background processes on your VPS and send alerts to any room you choose. You can create bots from the CLI or from the web admin panel.

Bot commands

xtc delete:bot all

Architecture

The bot wizard runs on your Mac/Linux machine (client). After configuration it sends a POST /bot/start to the server, which spawns bot_runner.py as a background process on the VPS. The bot then reads its config from the database and starts monitoring.

Mac VPS │ │ │ xtc start:bot │ │ (setup wizard) │ │ │ │─── POST /bot/register ───────→│ save config to db │─── POST /bot/start ──────────→│ spawn bot_runner.py │ │ │ │ bot monitors VPS... │ │ POST /send → alerts to room │ │ │ xtc stop:bot 1 │ │─── POST /bot/kill ───────────→│ kill PID, update db
ℹ️

bot_runner.py must be in the xtc-server/ folder on your VPS. Install psutil on the VPS for resource monitoring: pip3 install psutil

14 HTTPS setup

Caddy (recommended)

sudo apt install caddy -y

# /etc/caddy/Caddyfile
yourdomain.com {
    reverse_proxy localhost:8080
}

sudo systemctl reload caddy

Nginx + Let's Encrypt

sudo apt install nginx certbot python3-certbot-nginx -y
sudo certbot --nginx -d yourdomain.com
server {
    listen 443 ssl;
    server_name yourdomain.com;
    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;
    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
xtc connect @yourdomain.com

18 Troubleshooting

Connection refused

curl http://YOUR_IP:8080
sudo ufw status
cat server.pid && ps aux | grep server.py

Identity locked to another device

sqlite3 xtc.db "DELETE FROM users WHERE username='yourname';"

Port in use

sudo lsof -i :8080
sudo kill -9 <PID>
rm server.pid && python3 server.py start

Bot not starting

# Check bot_runner.py is in xtc-server/
ls xtc-server/bot_runner.py

# Check log
cat ~/.xtc_bot_1.log

# Make sure psutil is installed on VPS
pip3 install psutil

Web admin blank / 500 error

Make sure app.py uses send_from_directory (not render_template) — Flask's Jinja2 conflicts with HTML comment syntax in the files.

# Correct app.py pattern:
from flask import Flask, send_from_directory
@app.route('/dashboard')
def dashboard():
    return send_from_directory(html_dir, 'dashboard.html')

Copy not working (Linux)

sudo apt install xclip   # or xsel

19 FAQ

Can CLI and web users chat together?

Yes. Both use the same API and see each other's messages in real time in the same rooms.

Can I manage CLI-created bots from the web admin?

Yes. The web admin uses /bot/list/all to show all bots, and /bot/admin/kill / /bot/admin/delete to manage them using your web user PIN (which must be a registered user on the server).

Where does bot_runner.py need to be?

Always in xtc-server/ on the VPS — never on your local machine. The server spawns it directly.

Are messages stored permanently?

Until :purge (wipes messages, keeps room) or xtc delete:room (removes everything). Delete xtc.db for a full reset.

What if I switch machines?

Hardware PIN changes. Admin resets the old entry:

sqlite3 xtc.db "DELETE FROM users WHERE username='yourname';"

How do I update?

# Server
cd xtc-server && python3 server.py stop && git pull && python3 server.py start

# Client
cd xtc-client && git pull && make install
