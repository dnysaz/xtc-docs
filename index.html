# Introduction

XtermChat is a lightweight, self-hosted chat system built for the terminal. Unlike third-party platforms, XtermChat stores nothing on external servers. You deploy the Gateway on your own infrastructure, and all communication remains private, encrypted, and under your absolute control.

👤

Who is this for?

  • Sysadmins & Developers
  • Privacy-focused teams
  • Homelab enthusiasts
  • Internal secure comms

How it works

XtermChat Architecture Diagram:

┌─────────────────────────────────────────────────────────┐
│                      YOUR VPS                           │

│   ┌──────────┐   ┌──────────┐   ┌──────────┐            │

│   └──────────┘   └──────────┘   └──────────┘            │

                        │ REST API (JSON)
          ┌─────────────┼──────────────┐
          │             │              │
   ┌──────▼──────┐ ┌────▼──────┐  ┌────▼──────┐
   │  CLI Client │ │Web Browser│  │ CLI Client│

Backend: Python+Flask, Database: SQLite. The CLI uses rich and prompt_toolkit for the UI. Auth: Username + Hardware PIN (CLI) or 5-digit PIN (Web).

📋 Prerequisites

🖥️

Server requirements

  • Ubuntu 20.04+ / Debian 11+
  • Python 3.8+ & pip
  • Open port 8080 (TCP)
  • Min. 512 MB RAM
  • Optional: systemd (for auto-restart)
💻

Client requirements

  • macOS 11+ / Linux kernel 5.x / Windows 10+
  • Python 3.8+ and pip
  • Terminal with UTF-8 support
  • For Linux: xclip or xsel (optional, for copy)
  • Windows Terminal (recommended)
💡 Note: Windows users — please see the dedicated Windows section below.

⚙️ Server setup

1git clone https://github.com/dnysaz/xtc-server.git && cd xtc-server
2pip3 install flask flask-cors werkzeug
3sudo ufw allow 8080 && sudo ufw enable
4python3 server.py start

Auto‑creates xtc.db, server.log.

💾 Client setup

git clone https://github.com/dnysaz/xtc-client.git
cd xtc-client
make install   # (Mac/Linux – for Windows see below)
xtc

Configuration stored in ~/.xtc_config.json.

⊞ Windows Support (Native)

XtermChat runs natively on Windows — no WSL, no Docker required. Use it directly in PowerShell, CMD, or Windows Terminal.

🔧 Installation steps for Windows

  1. 1 Download and install Python 3.10+ from python.org (ensure "Add to PATH" is checked).
  2. 2 Install Windows Terminal from Microsoft Store (for best emoji/unicode support).
  3. 3 Open Command Prompt or PowerShell and run:
    git clone https://github.com/dnysaz/xtc-client.git
  4. 4 Enter folder and install dependencies:
    cd xtc-client & pip install -r requirements.txt
  5. 5 Run client: python xtc.py or create an alias.
✅ Clipboard integration works via clip / PowerShell Set-Clipboard. Emoji fully supported in Windows Terminal.

Identity & security

Hardware PIN: Username bound to Machine UUID. Only the original device can send as that user.

Encryption: Passwords hashed with scrypt. Room passwords never stored in plaintext.

API reference

EndpointMethodAuth Required
/roomsGETNo
/messages/<room>GETPassword (if set)
/sendPOSTValid PIN
/purge-chatPOSTCreator PIN
/delete-roomPOSTCreator PIN

Database Structure

  • users: (username PRIMARY, pin TEXT)
  • rooms: (name UNIQUE, creator, password, description, creator_pin)
  • messages: (room, sender, content, timestamp, pin)

FAQ

Messages permanent?

Only until Room Creator uses :purge.

Hardware Reinstall?

Admin must delete entry from users table if hardware ID changes.
