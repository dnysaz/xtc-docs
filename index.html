# Introduction

XtermChat is a lightweight, self-hosted chat system built for the terminal. Unlike third-party platforms, XtermChat stores nothing on external servers. You deploy the Gateway on your own infrastructure, and all communication remains private, encrypted, and under your absolute control.

👤

Who is this for?

  • Sysadmins & Developers
  • Privacy-focused teams
  • Homelab enthusiasts
  • Internal secure comms

Backend: Python+Flask, Database: SQLite. The CLI uses rich and prompt_toolkit for the UI. Auth: Username + Hardware PIN (CLI) or 5-digit PIN (Web).

Requirements

Server Side

  • Ubuntu 20.04+ / Debian 11+
  • Python 3.8+ & Pip
  • Open Port 8080 (Gateway)
  • Min. 512MB RAM

Client Side

  • macOS, Linux, or Windows
  • Python 3.8+
  • Internet connectivity to Server

Server setup

1git clone https://github.com/dnysaz/xtc-server.git && cd xtc-server
2pip3 install flask flask-cors werkzeug
3sudo ufw allow 8080 && sudo ufw enable
4python3 server.py start

Note: The server auto-generates xtc.db and server.log upon first boot.

Client setup

git clone https://github.com/dnysaz/xtc-client.git
cd xtc-client
make install   # (Mac/Linux)
xtc            # Show help

Configuration is persistently stored in ~/.xtc_config.json.

🪟 Windows Support

XtermChat is fully compatible with Windows. You do not need WSL or complex environments—standard Python is enough.

Installation for Windows:

  1. 1 Install Python 3.10+ from the official website.
  2. 2 Use Windows Terminal (recommended) instead of CMD.
  3. 3 Run python xtc.py connect @YOUR_IP to start.
⚠️ Note: Ensure your terminal supports UTF-8 for proper emoji and Rich UI rendering.

First time use

xtc connect @YOUR_VPS_IP:8080
xtc status
xtc create:room @general
xtc start:chat @general

CLI commands

CommandTargetDescription

Identity & security

Hardware PIN: XtermChat binds your username to your Machine UUID. Once registered, only the physical device that created the account can send messages as that user.

Encryption: Room passwords are never stored in plain text. We use scrypt hashing to ensure that even if the database is leaked, passwords remain secure.

Database Structure

XtermChat utilizes a single xtc.db (SQLite) file. Schema overview:

  • users: (username PRIMARY, pin TEXT)
  • rooms: (name UNIQUE, creator, password, description, creator_pin)
  • messages: (room, sender, content, timestamp, pin)

FAQ

Messages permanent?

Only until the Room Creator uses the :purge command.

Hardware Reinstall?

If your Hardware ID changes, an Admin must manually clear your entry from the users table.
