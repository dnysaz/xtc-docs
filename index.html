01 Introduction

XtermChat is a lightweight, self-hosted chat system built for the terminal. Unlike Slack, Discord, or Matrix вҖ” XtermChat stores nothing on third-party servers. You deploy the server on your own VPS, your team connects via the CLI client or web browser, and all messages stay on infrastructure you control.

рҹ‘Ө Who is this for?
  • Developers and sysadmins who prefer the terminal
  • Teams that need private internal communication
  • Anyone who does not want their data on someone else's servers
  • Homelab and self-hosted enthusiasts

Backend: Python+Flask, database SQLite. CLI uses prompt_toolkit, web ui served on port 5000. Auth: username + hardware PIN (CLI) or 5вҖ‘digit PIN (web).

03 Requirements

Server

  • Ubuntu 20.04+ / Debian 11+
  • Python 3.6+, pip
  • Port 8080 open
  • 512 MB RAM

CLI client

  • macOS or Linux
  • Python 3.6+, pip, make
  • Web client: any browser

04 Server setup

1 git clone https://github.com/dnysaz/xtc-server.git && cd xtc-server

2 pip3 install flask flask-cors werkzeug

3 sudo ufw allow 8080 && sudo ufw enable

4 python3 server.py start (background)

5 Verify: curl http://YOUR_IP:8080 вҶ’ {"service":"XtermChat Gateway"}

AutoвҖ‘created: xtc.db, server.log, server.pid.

05 Client setup 

git clone https://github.com/dnysaz/xtc-client.git
cd xtc-client
make install
xtc   # show help

Config saved to ~/.xtc_config.json.

06 First time use 

xtc connect @YOUR_VPS_IP:8080
xtc status
xtc create:room @general     # interactive or quick
xtc start:chat @general

09 Web interface

xtc start:web вҶ’ open http://localhost:5000. Use 5вҖ‘digit PIN (user chosen).

Web vs CLI: web PIN not deviceвҖ‘bound, uses localStorage.

10 Room management

Public (no password) or private (bcrypt). Creator can :purge messages and delete room.

11 Identity & security

CLI uses machine UUID (PIN). Username from getuser(), max 5 chars. Stored in users table.

12 HTTPS setup

Caddy / Nginx + Let's Encrypt. Then xtc connect @yourdomain.com.

13 Server management

python3 server.py start|stop|restart, logs: server.log. systemd example provided.

14 Database

Inspect with python3 check_db.py or sqlite3 xtc.db. Backup xtc.db.

16 Troubleshooting

  • [!] CONNECTION FAILED вҖ“ check firewall, VPS panel, curl
  • Identity locked вҖ“ delete from users table
  • Purge only works on creator machine (hardware PIN)
  • Copy on Linux: install xclip or xsel

17 FAQ

Multiple people in same room?
Yes, polling every 2s.
Messages permanent?
Until :purge or delete room.
Connect to multiple servers?
One active config, overwritten.
OS reinstall?
PIN changes; admin can delete user entry.
Update server/client?
git pull + restart / make install.
